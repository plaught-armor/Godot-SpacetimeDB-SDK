# Migrating from 0.2.x to 1.0

The 1.0 release targets **SpacetimeDB 2.0.0+** and uses the v2 BSATN binary protocol. It is not compatible with SpacetimeDB 1.x servers.

## Requirements

-   **SpacetimeDB 2.0.0+** (server and CLI)
-   Godot 4.4.1+ (same as 0.2.x)

## Breaking Changes

-   **SpacetimeDB 2.0.0+ required.** The SDK now uses the v2 BSATN protocol. Connecting to a SpacetimeDB 1.x server will not work.
-   **Removed signals:** `reducer_call_response` and `reducer_call_timeout` have been removed from `SpacetimeDBClient`. If you connected to these signals, see the migration steps below.
-   **`SpacetimeDBSubscription` is now `RefCounted`** instead of `Node`. The public API is unchanged (`subscribe()`, `applied`, `end`, `wait_for_applied()`, `unsubscribe()`), but subscription handles no longer appear in the scene tree.
-   **Query builder validates identifiers.** `SpacetimeDBQuery.where()` and `.table()` now reject field/table names containing special characters. Only alphanumeric characters and underscores are allowed.

## New Features

-   **Procedures:** SpacetimeDB 2.0 procedures are fully supported. Generated procedure wrappers are available via `SpacetimeDB.MyModule.procedures.my_proc()`, returning a `SpacetimeDBProcedureCall` handle with `wait_for_response()` and `decode()`.
-   **Deep Nesting:** Arbitrary nesting of `Option<T>` and `Vec<T>` types is now supported: `Option<Option<T>>`, `Vec<Vec<T>>`, `Option<Vec<Option<T>>>`, etc.
-   **Subscription Lifecycle:** The `end` signal now fires when the server confirms an unsubscribe. `SubscriptionErrorMessage` errors are propagated to the subscription handle via `error_message`. Subscription handles are properly invalidated on disconnect/reconnect.
-   **Subscription Error Handling:** `wait_for_applied()` now resolves immediately with `ERR_DOES_NOT_EXIST` if the subscription errors or ends, instead of timing out. Check `subscription.error_message` for details.

## Steps to Migrate

### 1. Update your SpacetimeDB server to 2.0.0+

The SDK uses the v2 BSATN protocol which requires SpacetimeDB 2.0.0 or later. Update your server and CLI before proceeding.

### 2. Regenerate your module bindings

Open your project in Godot, go to the SpacetimeDB tab, and click `Generate schema` to regenerate bindings. The codegen now produces procedure wrappers and handles deep nesting.

### 3. Remove `reducer_call_response` / `reducer_call_timeout` connections

These signals have been removed. If you were connecting to them:

```gdscript
# Before (0.2.x):
SpacetimeDB.MyModule.reducer_call_response.connect(_on_reducer_response)
SpacetimeDB.MyModule.reducer_call_timeout.connect(_on_reducer_timeout)

# After (1.0): Use the SpacetimeDBReducerCall handle directly
var call := SpacetimeDB.MyModule.reducers.my_reducer(arg1, arg2)
var result = await call.wait_for_response()
if call.is_error():
    print("Error: ", call.error_message)
elif call.outcome == SpacetimeDBReducerCall.Outcome.TIMEOUT:
    print("Timed out")
```

### 4. (Optional) Use the new subscription error handling

Subscription errors are now propagated instead of silently dropped:

```gdscript
var sub := SpacetimeDB.MyModule.subscribe(queries)
var err := await sub.wait_for_applied()
match err:
    OK:
        print("Subscription active")
    ERR_TIMEOUT:
        print("Timed out waiting for subscription")
    ERR_DOES_NOT_EXIST:
        print("Subscription failed: ", sub.error_message)
```

### 5. (Optional) Use procedures

If your SpacetimeDB 2.0 module defines procedures, they are now available:

```gdscript
var call := SpacetimeDB.MyModule.procedures.my_procedure(arg1, arg2)
var result_bytes := await call.wait_for_response()
if call.is_ok():
    var value = call.decode()  # Typed return value
```

### 6. (Optional) Use deep nesting

Fields with nested `Option`/`Vec` types now serialize correctly:

```gdscript
# Option<Vec<Option<String>>> works out of the box
row.nested_field = Option.some([Option.some("hello"), Option.none()])
```
