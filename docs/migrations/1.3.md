# Migrating from 1.2.x to 1.3.0

## Requirements

-   **SpacetimeDB 2.1.0+** (server and CLI)
-   Godot 4.4.1+

## Breaking Changes

-   **SpacetimeDB 2.1.0+ required.** Schema v9 support has been completely removed. The codegen now only fetches schema v10 (`?version=10`), which was introduced in SpacetimeDB 2.1.0. If you are running SpacetimeDB 2.0.x, you must upgrade before using this version of the SDK.

## Steps to Migrate

### 1. Update SpacetimeDB to 2.1.0+

```bash
# Check your current version
spacetimedb version

# Update to 2.1.0 or later
# See https://spacetimedb.com/install for installation instructions
```

### 2. Regenerate your module bindings

Open your Godot project, go to the SpacetimeDB dock, and click **Generate**. The codegen will fetch the v10 schema from your server. If you see an error about missing `sections`, your server is still on 2.0.x.

### 3. Restart Godot

After regenerating bindings, restart Godot to pick up the new generated classes.
